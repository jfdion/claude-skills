# Adaptation à la discipline - Guide de référence

Ce document aide à détecter la discipline d'un document et à adapter le vocabulaire des solutions suggérées.

---

## Détection de la discipline

### Indicateurs par discipline

#### Sciences naturelles (Chimie, Physique, Biologie)
**Mots-clés:** expérience, laboratoire, protocole, échantillon, mesure, données, observations, résultats, hypothèse, méthode expérimentale, analyse quantitative

**Livrables typiques:** rapport de laboratoire, cahier d'expérience, protocole expérimental, analyse de données

#### Sciences humaines et sociales (Sociologie, Psychologie, Histoire, etc.)
**Mots-clés:** analyse, recherche, théorie, participants, enquête, entrevue, questionnaire, échantillonnage, méthodologie qualitative, revue de littérature, cadre théorique

**Livrables typiques:** essai, analyse comparative, revue de littérature, rapport de recherche, étude de cas

#### Arts et lettres (Littérature, Arts visuels, Musique, etc.)
**Mots-clés:** œuvre, création, analyse, critique, démarche artistique, portfolio, interprétation, style, technique, composition, performance

**Livrables typiques:** analyse d'œuvre, portfolio, carnet de création, essai critique, projet créatif

#### Santé (Soins infirmiers, Médecine, Santé publique)
**Mots-clés:** patient, soins, diagnostic, intervention, plan de soins, clinique, symptômes, traitement, évaluation, protocole clinique, cas clinique

**Livrables typiques:** plan de soins, analyse de cas clinique, rapport d'intervention, protocole de soins

#### Gestion et administration (Marketing, Finance, RH)
**Mots-clés:** entreprise, marché, stratégie, analyse financière, plan d'affaires, clientèle, budget, ressources humaines, gestion de projet, étude de faisabilité

**Livrables typiques:** plan d'affaires, analyse de marché, rapport financier, étude de faisabilité, plan stratégique

#### Techniques et ingénierie
**Mots-clés:** système, conception, spécifications, normes, procédure, installation, maintenance, schéma, diagramme, cahier des charges, tests, validation

**Livrables typiques:** rapport technique, cahier des charges, manuel d'utilisation, documentation technique, plan de conception

#### Informatique et technologies
**Mots-clés:** code, programmation, algorithme, architecture, système, développement, base de données, réseau, application, logiciel, documentation technique

**Livrables typiques:** code source, documentation technique, rapport de projet, analyse de systèmes, cahier d'analyse

## Application dans le skill

### Lors de la détection de discipline

1. **Analyser le vocabulaire dominant** dans le document
2. **Identifier les livrables mentionnés** (rapport de labo, essai, portfolio, etc.)
3. **Détecter le contexte pédagogique** (cours de chimie, atelier d'art, stage clinique, etc.)
4. **Classifier dans une discipline principale** (avec sous-discipline si pertinent)

### Lors de la génération des solutions

1. **Utiliser le vocabulaire spécifique** à la discipline détectée
2. **Adapter les exemples** aux types de livrables typiques
3. **Référencer les standards** pertinents à la discipline (normes, protocoles, méthodes)
4. **Maintenir la cohérence** dans toutes les suggestions du tableau

### Si discipline incertaine

- Utiliser un vocabulaire **générique et inclusif**
- Mentionner dans le résumé: "Discipline: Générique (discipline non clairement identifiée)"
- Proposer des solutions applicables à multiple contextes

## Notes importantes

- **L'adaptation est AUTOMATIQUE** - le skill détecte et s'adapte sans intervention
- **Le vocabulaire doit rester clair** - éviter le jargon excessif même dans les solutions
- **La discipline détectée est indicative** - en cas de doute, rester générique
- **Les solutions restent des suggestions** - l'enseignant décide de l'application finale
